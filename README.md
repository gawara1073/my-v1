# コマ合わせ 📅 | 大学生向け空き時間チェッカー

LINEでの面倒な予定調整を、タップだけで一瞬で解決する大学生向け「空き時間チェッカー」の動作プロトタイプv1です。
サーバー不要、HTML/CSS/JavaScriptのみで動作し、**GitHub Pages** にデプロイするだけで即座に運用できます。

---

## 🌟 特長

1. **サーバー不要！完全フロントエンド動作**
   - データベースやサーバーを用意する必要がありません。静的ホスティング（GitHub Pages、Vercelなど）で今すぐ無料で公開できます。
2. **P2PライクなURL共有システム（LINE共有）**
   - 各自のスケジュールデータをBase64エンコードし、URLパラメータとして埋め込みます。LINEでURLを送り合うだけで、自動的に予定が重ね合わされます。
3. **スマホファースト＆超直感UI/UX**
   - 大学生が使い慣れた「曜日×時限（1限〜6限）」の時間割グリッドを採用。
   - 指でなぞる（ドラッグ/スワイプ）だけで、空いているコマをスムーズに連続選択できます。
4. **リアルタイム・ヒートマップ**
   - メンバー全員の空き時間を重ね合わせて「ヒートマップ」として自動描画。
   - 誰が空いているかはセルをタップするだけでポップアップ表示されます。
5. **AI風おすすめ日程の自動提案**
   - 登録メンバーの空き状況を自動計算し、「全員が空いているコマ」や「最も人が集まるコマ」を自動で提案します。
6. **ローカルデモシミュレーション**
   - 1台の端末でも使い勝手を試せるよう、その場で「Aさん」「Bさん」とメンバーを追加してシミュレーションできる「メンバーに追加する」機能を搭載しています。

## 🚀 使い方（LINEでの予定調整の流れ）

1. **主催者が予定を作る**
   - 「コマ合わせ」のウェブサイトを開きます。
   - 自分の名前（例: `たろう`）を入力します。
   - カレンダーから空いているコマをタップ（または指でなぞって選択）します。
   - 「メンバーに追加する」を押し、自分のデータを確定させます。
   - 「LINEに共有URLをコピー」ボタンを押し、生成されたURLをLINEグループに送信します。

2. **友達が予定を重ねる**
   - 友達がLINE内のリンクを開くと、**すでに「たろう」さんの空き時間が読み込まれた状態**でスタートします。
   - 友達も自分の名前（例: `はなこ`）を入力し、空いているコマを選択して「メンバーに追加する」を押します。
   - 「LINEに共有URLをコピー」ボタンを押し、新しい全員分のデータが含まれたURLを再度LINEグループに送信します。

3. **最適な日程を見つける**
   - メンバーが増えるたびに、結果エリアのヒートマップの緑色が濃くなり、おすすめ時間帯がリアルタイムに更新されます。
   - 全員が空いている「超おすすめ」コマや、最も集まりやすいコマが一目で分かり、LINEでのグダグダな調整から解放されます！

## 🌐 GitHub Pages への公開手順

このプロトタイプは、以下の手順で簡単にインターネット上に無料公開できます。

1. **GitHubリポジトリを作成する**
   - GitHubにログインし、新しいリポジトリ（例: `koma-awase`）を作成します。

2. **コードをプッシュする**
   - ローカルPCの `my-v1` ディレクトリの中身をGitリポジトリにコミットし、GitHubへプッシュします。
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin <あなたのGitHubリポジトリURL>
   git push -u origin main
   ```

3. **GitHub Pages を有効化する**
   - リポジトリの **[Settings]** タブを開きます。
   - 左側メニューの **[Pages]** を選択します。
   - **Build and deployment** の **Branch** で `main` (または `master`) を選択し、フォルダを `/ (root)` にして **[Save]** をクリックします。
   - 数分後、`https://<ユーザー名>.github.io/koma-awase/` にてアプリが公開されます！

## 🎨 技術的なポイント

- **Base64日本語エスケープ**: 
  JavaScriptの標準の `btoa()` / `atob()` では日本語（マルチバイト文字）で文字化けが発生するため、UTF-8エンコードを経由して安全にシリアライズ・デシライズしています。
- **スマホなぞり操作の実現**:
  スマートフォンでは `touchmove` 中に `event.target` がドラッグ開始時のセルに固定されるため、タッチ座標 `clientX` / `clientY` から `document.elementFromPoint()` を使用して現在指が触れているセルを特定し、動的トグルを実現しています。
- **ヒートマップの表現**:
  CSS変数および `rgba` のアルファ値（透明度）を利用することで、何％の人が空いているかを色強度として動的にマッピングしています。
